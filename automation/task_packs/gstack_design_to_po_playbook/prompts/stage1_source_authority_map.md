# Stage 1 - Source Authority Map

Read the primary job inputs and produce a source authority map for a future `markdown_playbook_v1` playbook.

Output must include:

1. the authoritative source files supplied by the operator and their relative authority;
2. confirmed implementation goals;
3. explicit non-goals;
4. unresolved ambiguities that must become manual gates or exclusions;
5. claims that are not sufficiently supported by attached evidence;
6. a first-pass list of phases, without execution rows.

Do not write a final playbook in this stage.
