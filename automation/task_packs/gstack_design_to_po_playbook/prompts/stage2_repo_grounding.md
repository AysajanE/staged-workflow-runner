# Stage 2 - Repository Grounding

Using the approved Stage 1 review bundle and the attached repository context, ground the plan in concrete repo surfaces.

Output must include:

1. relevant source, test, docs, config, package, and CI files;
2. likely write roots, with least-privilege rationale;
3. repo-native verification commands;
4. manual-gate triggers for secrets, migrations, production data, external services, security boundaries, compliance claims, or irreversible operations;
5. unsupported assumptions that must not appear as facts in the playbook.

Do not write a final playbook in this stage.
